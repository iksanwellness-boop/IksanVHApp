🚀 Dar acceso a un nuevo consultor
Sigue estos pasos para activar un nuevo usuario
1
Crea el usuario en Suscripciones
Ve a 💳 Suscripciones → llena nombre, email, contraseña, selecciona módulos y periodo.
2
Comparte el archivo HTML con el consultor
Descarga y envía el mismo archivo iksan_wellness_app_3.html al consultor por WhatsApp, email o Drive. Sus datos quedan en su propio dispositivo.
3
El consultor abre el archivo y entra con sus credenciales
El archivo corre en cualquier navegador sin instalación. Sus datos (consultantes, ventas, citas) quedan guardados en su dispositivo y son sólo suyos.
4
El consultor configura su marca (opcional)
En BO Consultor → 🎨 Mi Config puede poner su logo, colores y nombre de negocio. Se aplica solo en su dispositivo.
⚠️ Limitación actual (localStorage)
Actualmente cada consultor que abra el mismo archivo en el mismo navegador comparte los datos. Para que cada consultor tenga datos completamente separados, cada uno debe abrir el archivo en su propio dispositivo o navegador.

Para SaaS real con datos en la nube (múltiples consultores con login compartido y datos separados) se necesita un servidor. Las mejores opciones son:
🥇 Opción A — Supabase (recomendada)
Base de datos PostgreSQL gratuita en la nube. Plan gratis hasta 500MB. Cada clave de localStorage se guarda por userId. Migración ~2 días de desarrollo.
🥈 Opción B — Firebase (Google)
Firestore en tiempo real. Gratis hasta 1GB. Auth incluido. Más sencillo de implementar para apps sin servidor.
🥉 Opción C — GitHub Pages + archivo por consultor
Sube el HTML a GitHub Pages (gratis). Cada consultor lo abre en su dispositivo. Sin servidor, sin migración. Ideal para empezar hoy.
